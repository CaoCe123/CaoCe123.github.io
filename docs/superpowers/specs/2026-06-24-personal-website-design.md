# 个人网站 — 设计文档

**日期**：2026-06-24
**项目**：一个公开可访问的个人网站（零基础用户，全程带做）

---

## 1. 目标与背景

为一位前端零基础的用户做一个**公开的个人网站**，包含四个板块：
- **A. 个人简介**（我是谁）
- **B. 作品集 / 简历**（展示作品、项目、技能）
- **C. 博客**（持续添加文章）
- **D. 爱好展示**（兴趣、收藏等）

用户审美偏好：**有趣、有个性、有记忆点**（已选定视觉方向 C）。

成功标准：网站做出来后能**免费部署上线**，拿到一个任何人都能打开的公开网址。

---

## 2. 技术选型

**纯静态网站：HTML + CSS + 原生 JavaScript，不使用框架或构建工具。**

理由：
- 用户零基础，静态文件可直接用浏览器打开预览，无安装/编译/配置环节。
- 便于用户日后自行用文本编辑器修改。
- 免费托管平台（GitHub Pages / Netlify / Vercel）对静态站点支持最好，部署最简单。

明确**不做**（YAGNI）：不引入 React/Vue、不引入打包工具、不引入后端、博客暂不引入 CMS/静态站点生成器。

---

## 3. 网站结构

**主体为单页滚动长首页 + 每篇博客文章独立成页。**

```
个人网站/
├── index.html          # 主页（单页滚动，含下列所有板块）
├── style.css           # 全站样式（颜色、布局、动效）
├── script.js           # 趣味交互（滚动浮现、鼠标效果、平滑滚动、回到顶部）
├── posts/              # 博客文章
│   ├── _template.html  # 写新文章的模板（复制填空）
│   └── (各篇文章).html
├── images/             # 头像、作品图、照片等
└── 发文章小抄.md        # 给用户的「发新文章三步走」中文说明
```

---

## 4. 主页板块（从上到下）

1. **顶部导航条** — 名字/Logo + 导航项（首页 / 作品 / 爱好 / 博客 / 关于）。点击平滑滚动到对应板块；滚动时高亮当前所在板块。移动端导航收起为简化形式。
2. **Hero 开场（板块 A 门面）** — 流动渐变背景光斑 + 渐变大标题「我是 XXX」+ 一句有个性的自我介绍 + 主按钮（进入作品/联系我）+ 鼠标有反应的小彩蛋（标题对鼠标位置有轻微反应）。
3. **作品集（板块 B）** — 卡片网格，每张卡片含缩略图/标题/简短描述；鼠标悬停有俏皮的微交互（轻微放大/倾斜/发光）。
4. **我的爱好（板块 D）** — 图文混排展示兴趣（如摄影/画画/收藏）。
5. **博客（板块 C）** — 列出最近若干篇文章（标题 + 日期 + 一句摘要），点击进入 `posts/` 下对应文章页阅读全文。
6. **关于我 & 联系方式（板块 A 收尾）** — 较完整的自我介绍 + 社交链接 / 邮箱。
7. **页脚** — 版权信息 + 回到顶部。

---

## 5. 博客机制（手动填模板方式）

- `posts/_template.html` 是单篇文章模板，包含统一的头部导航/返回链接/样式引用，正文区域留空待填。
- 用户发新文章流程（写入 `发文章小抄.md`）：
  1. 复制 `_template.html`，重命名为新文件（如 `my-trip.html`）。
  2. 用文本编辑器填入标题、日期、正文。
  3. 在 `index.html` 博客板块中新增一行链接指向该文件。
- 模板与小抄都要做到**对零基础用户足够友好**（带注释、示例、明确指出"在这里填字"）。

明确**不做**：自动文章索引、标签系统、评论、RSS（后续可升级）。

---

## 6. 视觉与交互设计（方向 C：大胆有趣）

- **配色**：深色背景 `#0d0d0f`；紫(`#7c3aed`)→青(`#06b6d4`)渐变作为点缀与按钮；浅色文字，正文 `#c7c7d6` 级别保证对比度。
- **字体**：标题 **Archivo**，正文 **Space Grotesk**（Google Fonts，`display=swap`）。
- **趣味动效**：
  - 缓慢流动的渐变背景光斑。
  - 滚动时内容分块带回弹地浮现（Intersection Observer）。
  - 卡片/按钮悬停微交互（轻微放大/倾斜/发光，150–300ms）。
  - Hero 标题对鼠标位置有轻微反应（小彩蛋，点到为止）。
  - 圆润胶囊形按钮，渐变填充。
- **无障碍**：尊重 `prefers-reduced-motion`（开启时关闭/减弱动画）；文字对比度 ≥ 4.5:1；图标用 SVG 而非 emoji；可键盘导航，焦点可见。
- **响应式**：移动优先，在 375 / 768 / 1024 / 1440 断点下均可用，无横向滚动。

---

## 7. 部署上线（网站完成后进行）

使用 **GitHub Pages 或 Netlify** 免费托管，获得公开网址（如 `用户名.github.io`）。此步骤在网站本体完成后，由助手手把手带用户操作（含注册账号、上传文件、拿到网址）。

---

## 8. 需用户后续提供的真实内容（可先用占位）

- 名字 / 昵称、一句话自我介绍、较完整的个人介绍。
- 作品 / 项目（标题 + 简介 + 图片）。
- 爱好板块内容。
- 社交链接 / 邮箱。
- 头像及图片素材。

策略：**先用占位内容把网站整体跑通并可预览**，再逐步替换为真实内容。

---

## 9. 验收

- 本地用浏览器打开 `index.html`，四个板块齐全、导航可平滑跳转、趣味动效正常、移动端不破版。
- 博客模板可复制填出一篇可正常打开的文章页。
- `发文章小抄.md` 步骤清晰，零基础可照做。
- （最终）成功部署，公开网址可被他人打开。
